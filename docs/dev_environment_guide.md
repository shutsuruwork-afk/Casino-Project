# 🛠️ GameHub 開発ツール・使い方ガイド

Codespaceのリビルド完了後、Cursor（VS Code）には以下のツール群が全自動でインストールされ、すぐに使えるようになっています。それぞれの「何ができるか」「どう使うか」のまとめです。

---

## 1. 🔍 コーディング支援（全自動・操作不要）
これらは裏で勝手に動くため、あなたが**特別な操作をする必要はありません**。

*   **Rust Analyzer / Go**
    *   コードを打つたびにリアルタイムで文法チェックや型の補完をしてくれます。
*   **ESLint / Prettier (Next.js用)**
    *   ファイルを「保存 (`Ctrl+S`)」した瞬間に、コードの改行やインデント（空白）をルールに従って**自動で綺麗に整頓**してくれます。
*   **Error Lens**
    *   エラーが出た時、波線にマウスを乗せなくても**コードの右端に赤文字で直接エラー内容が表示**されます。バグ修正が爆速になります。
*   **DotENV / YAML**
    *   `.env` ファイルや `docker-compose.yml` などの設定ファイルを開くと、見やすく綺麗に色分け（シンタックスハイライト）されます。

---

## 2. 📝 タスク管理： Todo Tree
コードの中に書き込んだ「あとでやる」メモを、プロジェクト全体から一瞬でかき集めてリスト化してくれます。

*   **使い方**:
    1. コードのどこかに `// TODO: ログイン機能を追加する` や `// FIXME: ここの計算バグってる` とコメントを書きます。
    2. 左のメニューバーにある **木のアイコン（Todo Tree）** をクリックします。
    3. プロジェクト内の全ファイルから `TODO` と `FIXME` が一覧で表示され、クリックするとその行にジャンプできます。

---

## 3. 🌐 APIテスト： REST Client
画面（フロントエンド）を作る前に、RustやGoのサーバーが正しく動いているかテストするツールです。

*   **使い方**:
    1. プロジェクト内に `test.http` という適当な名前のファイルを作ります。
    2. そのファイルに以下のように書きます。
       ```http
       GET http://localhost:8081/api/status
       ```
    3. 書いたURLの上に **`Send Request`** という小さな文字（ボタン）が浮かび上がるので、それをクリックすると右側にサーバーからの返答（JSONなど）が表示されます。

---

## 4. 🔌 通信テスト： WebSocket Test Client
カジノゲームの命である「リアルタイム通信（WebSocket）」をテストします。

*   **使い方**:
    1. `Ctrl + Shift + P` でコマンドパレットを開きます。
    2. **`WebSocket Test Client: Open`** と入力してEnterを押します。
    3. 専用のテスト画面が開くので、`ws://localhost:8080/ws` などを入力してConnectを押し、JSONメッセージを送信してテストします。

---

## 5. 🗄️ データベース操作： SQLTools
データベース（PostgreSQL）の中身（ユーザー情報やチップ残高）を直接覗き見したり、書き換えたりできます。

*   **使い方**:
    1. 左のメニューバーにある **ドラム缶のようなアイコン（SQLTools）** をクリックします。
    2. 「Add New Connection」から PostgreSQL を選び、パスワード等を入力して接続します。
    3. VS Code上で直接 `SELECT * FROM users;` などのSQLを打ってデータを表形式で確認できます。

---

## 6. 🧪 テスト網羅率： Coverage Gutters
自分の書いたテストコードが、全体のプログラムの「何％をカバーできているか」を可視化します。

*   **使い方**:
    1. エディタの一番下にある青いステータスバーに **`Watch`** という文字が出るのでクリックします。
    2. コードの左端（行番号の横）に、テスト済みの行は「緑」、テストされていない行は「赤」でラインが引かれます。（※テスト実行後に生成されるlcovファイルが必要です）
